CONTEXT: D4 audit found ATR exhaustion ratio mismatches for IBIT LP2 (+$4,192)
and BA MODEL4 (+$2,062). The audit used bar.close to approximate ATR distance,
but the filter chain uses signal.level.price.

ROOT CAUSE OF MISMATCH:
  The D4 audit approximation uses:
    distance = bar.close - day_low  (shorts)
    distance = day_high - bar.close (longs)

  The actual filter chain (_calc_atr_ratio) uses:
    distance = signal.level.price - day_low  (shorts)
    distance = day_high - signal.level.price (longs)

  Since bar.close != signal.level.price in general, the ratios differ.
  This is an audit approximation error, NOT a filter chain bug.

IMPACT: None. The ATR filter is DISABLED for the recommended Variant A'
config (enable_atr_filter=False). ATR ratios are informational only.
